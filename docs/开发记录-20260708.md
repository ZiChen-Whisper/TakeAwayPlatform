### 阶段一：用户与鉴权模块

**新增文件：**
- `src/middleware/auth_middleware.h` — JWT Token 验证 + RBAC 角色校验
- `src/services/user_service.h` — 用户注册/登录/个人信息 CRUD

**实现要点：**
- 注册时检查用户名唯一性
- 密码使用 SHA-256 + 随机盐值哈希存储（格式：`salt$hash`）
- 登录成功后生成 JWT Token（包含 user_id、role、exp 过期时间）
- 鉴权中间件从 `Authorization: Bearer <token>` 头提取并验证 JWT
- RBAC 支持 0=用户、1=商家、2=管理员 三级角色

**路由：**
| 方法 | 路径 | 鉴权 |
|------|------|------|
| POST | `/api/user/register` | 无 |
| POST | `/api/user/login` | 无 |
| GET | `/api/user/profile` | Token |
| PUT | `/api/user/profile` | Token |

### 阶段三：购物车模块

**新增文件：**
- `src/services/cart_service.h` — 加购/改量/删除/总价计算

**实现要点：**
- "一用户一购物车"模型，首次加购自动创建 cart 记录
- 加购时锁定菜品当前价格（价格快照）
- 总价使用 C++ 内存聚合计算（非 SQL 聚合），`round(total * 100) / 100.0` 保留两位小数
- 加购前检查菜品上架状态和库存

**路由：**

### 阶段五：订单与支付模块（核心模块）

**新增文件：**
- `src/utils/order_state_machine.h` — 订单状态机
- `src/services/order_service.h` — 订单创建/支付/查询

**状态机定义：**
```
待支付(0) ──支付成功──→ 待接单(1)
待支付(0) ──支付失败──→ 已取消(4)
待接单(1) ──商家接单──→ 配送中(2)
待接单(1) ──商家拒单──→ 已取消(4)
配送中(2) ──确认完成──→ 已完成(3)
已完成(3) → 终态
已取消(4) → 终态
```

**创建订单核心流程（事务）：**
1. 查询并校验购物车明细（菜品上架、库存充足）
2. `SELECT ... FOR UPDATE` 悲观锁锁定菜品行（防超卖）
3. 扣减库存 `UPDATE dish SET stock = stock - ?`
4. 创建订单主记录 + 订单明细
5. 地址信息 JSON 快照存储
6. 清空购物车
7. 提交事务 / 异常回滚

**模拟支付：**
- 90% 概率支付成功 → 状态变为"待接单"，记录 pay_time
- 10% 概率支付失败 → 状态变为"已取消"，**恢复库存**

**路由：**
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/orders` | 提交订单 |
| POST | `/api/orders/{id}/pay` | 模拟支付 |
| GET | `/api/orders` | 订单列表（支持状态筛选、分页） |
| GET | `/api/orders/{id}` | 订单详情 |

## 三、数据库设计

10 张数据表，全部使用 InnoDB 引擎，utf8mb4 字符集：

| 表名 | 说明 | 关键约束 |
|------|------|----------|
| `user` | 用户表 | username UNIQUE |
| `merchant` | 商家表 | user_id UNIQUE, FK→user |
| `category` | 分类表 | FK→merchant |
| `dish` | 菜品表 | FK→merchant, FK→category |
| `cart` | 购物车表 | user_id UNIQUE, FK→user |
| `cart_item` | 购物车明细 | FK→cart, FK→dish |
| `address` | 地址簿 | FK→user |
| `order` | 订单表 | FK→user, address_snapshot JSON |
| `order_item` | 订单明细 | FK→order |
| `system_config` | 系统配置 | config_key UNIQUE |

## 五、已知问题

1. **搜索结果菜品名称显示异常**：`d.*` 与 `c.name as category_name` 列别名冲突，需进一步调试 parse_result 的列名处理逻辑。不影响核心业务流程。
2. **Git push 超时**：网络/凭据问题，需手动 `git push`。
3. **libcrypto 版本警告**：MySQL Connector 使用 libcrypto.so.1.1，系统安装 libcrypto.so.3，存在潜在冲突，当前不影响运行。

## 六、部署说明

```bash
# 编译
cd ~/TakeAwayPlatform/build
cmake ..
make -j$(nproc)

# 部署
sudo cp TakeAwayPlatform /opt/TakeAwayPlatform/bin/
sudo cp ../config/config.json /opt/TakeAwayPlatform/config/

# 初始化数据库
mysql -u root -p < ../docs/database_init.sql

# 运行
./TakeAwayPlatform /opt/TakeAwayPlatform/config/config.json

# 或从源码目录直接运行
./TakeAwayPlatform ../config/config.json
```
